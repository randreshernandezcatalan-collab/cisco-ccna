Fundamentos del Protocolo
IGP de estado de enlace basado en el algoritmo SPF de Dijkstra.
Construye una LSDB y calcula la mejor ruta por costo.
Convergencia rápida, jerarquía por áreas y soporte ECMP.
Usa Router ID de 32 bits (formato IP). Se elige por loopback o manual.
Vecindad y Estados OSPF
Down → Init → 2-Way → ExStart → Exchange → Loading → Full
Hello y Dead timers deben coincidir para formar adyacencia.
En redes multiaccess aparecen DR y BDR para reducir flooding.
show ip ospf neighbor
show ospfv3 neighbor
Diseño, Áreas y Buenas Prácticas
Area 0 es el backbone; todas las demás áreas deben conectarse lógicamente a ella.
ABR conecta áreas; ASBR redistribuye rutas externas.
Passive-interface en enlaces LAN sin vecinos para seguridad y estabilidad.
Costos consistentes, router-id explícito y sumarización en bordes.

OSPFv2 — IPv4 y Comandos IOS
Proceso clásico para IPv4. Se anuncia con network + wildcard y area.
router ospf 1
router-id 1.1.1.1
auto-cost reference-bandwidth 1000
network 10.0.0.0 0.0.0.255 area 0
passive-interface g0/0
default-information originate
show ip ospf neighbor / interface / database
Comandos Transversales y Verificación
show ip route ospf
show ipv6 route ospf
show ip protocols
show ip ospf interface brief
show ospfv3 interface brief
show ip ospf database
show ospfv3 database
OSPFv3 — IPv6 y Comandos IOS
Diseñado para IPv6; la activación suele hacerse por interfaz.
ipv6 unicast-routing
ipv6 router ospf 10
router-id 3.3.3.3
auto-cost reference-bandwidth 1000
int g0/0
ipv6 address 2001:db8:1::1/64
ipv6 ospf 10 area 0
show ospfv3 neighbor / interface / database
